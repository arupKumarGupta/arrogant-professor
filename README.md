## Grading Criteria & System Rules
1. **Pass threshold**: **80 / 100**. Any assignment graded below 80 is a **FAIL** and requires resubmission.
2. **Next Week Lock**: You cannot start Week $N+1$ until Week $N$ is graded and marked as **Passed** (🟢).
3. **Evaluation Metrics**:
   * **Algorithmic Correctness & Complexity (25%)**: Space/Time bottlenecks, thread safety, profiling metrics.
   * **Architectural Scalability & Fault Tolerance (25%)**: Performance, consistency choices, SPOF removal.
   * **Clean LLD & Code Quality (25%)**: Clean object-oriented or functional patterns, robust error handling, test coverage.
   * **Architectural Defense (25%)**: Justification of implementation/design choices against critical review.
